## 5. GitHub Copilot ガイドライン策定

### 背景

- GitHub Copilot の導入を進めるにあたり、セキュリティ・ライセンス・品質の観点でルールが必要
- 開発者が安心して使える環境を整備する

### ガイドラインの主要項目（案）

| 項目 | 内容 |
|------|------|
| 利用対象者 | 対象部門・チーム・ライセンス割当の基準 |
| 許可される用途 | コード補完、テスト生成、ドキュメント補助 等 |
| 禁止事項 | 機密情報・個人情報をプロンプトに含めない、生成コードの無検証でのコミット禁止 等 |
| コードレビュー | Copilot 生成コードは通常のコードレビュープロセスを必ず通す |
| ライセンス | OSS ライセンス汚染のリスクと対策（Copilot のフィルタ設定） |
| セキュリティ | Copilot Business/Enterprise の利用（テレメトリ制御）、Suggestions matching public code の無効化 |
| 教育 | 利用開始前のオンボーディング資料・ハンズオンの実施 |

### 策定スケジュール

| フェーズ | 内容 | 時期 |
|---------|------|------|
| ドラフト作成 | セキュリティ・法務と連携しガイドライン初版を作成 | 3月末 |
| レビュー | 関係部門（情シス・法務・開発リード）のレビュー | 4月上旬 |
| パイロット | 一部チームで試験運用＆フィードバック収集 | 4月中 |
| 正式公開 | 全社向けに公開・周知 | 5月 |

## 6. 効果測定の方法

### パイロット期間中に計測する指標

| 指標 | 計測方法 | 頻度 |
|------|---------|------|
| Copilot 受け入れ率 | Copilot のダッシュボード（Acceptance Rate） | 週次 |
| タスク完了時間の変化 | Jira / チケット管理ツールの実績比較（導入前 vs 後） | スプリント毎 |
| コード量・PR頻度 | GitHub の PR 統計 | 週次 |
| 開発者アンケート | 体感の生産性・満足度を5段階で回答 | パイロット前後 |
| コードレビュー指摘数 | PR レビューコメント数の変化 | 月次 |

### 効果レポートの活用

- パイロット結果をもとに **全社展開の判断材料** とする
- 削減効果が数値で示せれば、**経営層への予算申請・拡大提案** に直結
