## 各モデルの詳細プロファイル

### 1. Gartner AI Maturity Model

- **提唱主体**: Gartner, Inc.
- **モデル名称**: AI Maturity Model（AI Maturity & Roadmap Toolkit）
- **業界**: 業界非依存（横断）
- **レベル数**: 5
- **各レベルの定義**:
  1. **Foundational / Awareness**: AIの可能性は認識しているが、戦略なし。アドホックな議論のみ
  2. **Active / Emerging**: 一部部署でPoCや実験。エンスージアスト主導、一貫した戦略なし
  3. **Operational**: 特定業務にAIが組み込まれ、エグゼクティブスポンサーと予算がある状態
  4. **Systemic / Scaled**: 全社業務の大半がAI対応。新規プロジェクトはAI利用を前提に検討
  5. **Transformational**: AIが事業DNAに組み込まれ、競争優位の中核
- **評価軸**:
  - ① 業務プロセス統合度: アドホック → 特定業務組み込み → 大半の業務がAI起動
  - ② 組織展開度: 個人/小グループ → 部署 → 部門横断 → 全社 → エコシステム
  - ③ 自動化レベル: 支援 → 部分自動化 → ML本番運用 → 自律化
  - ④ データ活用度: 散在データ → データ基盤整備 → 全社データドリブン
  - ⑤ ガバナンス成熟度: ルールなし → 標準策定 → ガバナンス内製化 → 継続改善
- **特徴**: 7つの柱（戦略・データ・ガバナンス・エンジニアリング・運用モデル・人材文化・AIプロダクト価値）でバランスよく評価。最もメジャーで業界調査でも参照基準として使われる
- **想定ユースケース**: 大企業のAI戦略立案、ロードマップ策定、ベンチマーク

### 2. Microsoft AI Maturity Model（旧来型）

- **提唱主体**: Microsoft
- **モデル名称**: AI Maturity Levels (Foundational / Approaching / Aspirational / Mature)
- **業界**: 業界非依存
- **レベル数**: 4
- **各レベルの定義**:
  1. **Foundational**: 認識はあるがほぼ未実装。AIへの懐疑も残る
  2. **Approaching**: 定型タスク自動化が始まり、業務効率改善が見える
  3. **Aspirational**: AIが戦略的役割。新規収益源やCX改善に貢献
  4. **Mature**: AIが事業戦略のコア。予測分析・パーソナライズ・新市場創出
- **評価軸**:
  - ① 業務プロセス統合度: 局所自動化 → 戦略的活用 → 中核機能化
  - ② 組織展開度: 個人 → 部門 → 全社
  - ③ 自動化レベル: 支援 → 定型自動化 → 戦略的活用 → 全社最適化
  - ④ データ活用度: 部分整備 → データ基盤整備 → 全社データ活用
  - ⑤ ガバナンス成熟度: 文化未対応 → ポリシー策定 → 倫理組織内蔵
- **特徴**: シンプルで4段階。各レベル間の "ジャンプ" の意味付け（運用→戦略、戦略→中核）が明確
- **想定ユースケース**: 経営層向け説明、AI投資の優先順位決定

### 3. Microsoft Agentic AI Adoption Maturity Model

- **提唱主体**: Microsoft（Copilot Studio Guidance）
- **モデル名称**: Agentic AI Adoption Maturity Model
- **業界**: 業界非依存（エージェント特化）
- **レベル数**: 5（Level 100, 200, 300, 400, 500）
- **各レベルの定義**:
  1. **Level 100 — Initial**: 個別の実験。一貫性なく、一部ユーザーが試行
  2. **Level 200 — Repeatable**: パターンが出現するも非公式
  3. **Level 300 — Defined**: ガバナンス・標準・運用モデルが整備
  4. **Level 400 — Capable**: エージェントがエンタープライズ計画・運用に組み込まれる
  5. **Level 500 — Efficient**: エージェント・ファースト企業として機能。継続最適化
- **評価軸**:
  - ① 業務プロセス統合度: 単発ユースケース → ワークフロー組み込み → 全社プロセス再構築
  - ② 組織展開度: 個人 → 部門 → 全社+パートナー
  - ③ 自動化レベル: 支援 → 部分自動化 → 自律エージェント → エージェント・オーケストレーション
  - ④ データ活用度: スポット利用 → セキュアな共有データ → リアルタイム共通基盤
  - ⑤ ガバナンス成熟度: なし → 標準策定 → 全社ガバナンス → 責任あるAI運用 → 継続改善
- **特徴**: 5本柱（戦略/プロセス変革/ガバナンス/技術データ/組織文化）でAIエージェントに特化。CMMI類似の番号体系
- **想定ユースケース**: AIエージェント／コパイロット／自律システムの導入計画

### 4. Microsoft GenAIOps Maturity Model

- **業界**: 業界非依存（LLM運用特化）

  - ⑤ ガバナンス成熟度: なし → 安全性指標 → コンテンツセーフティ → 包括的ガードレール

### 5. Google Cloud AI Adoption Framework

- **提唱主体**: Google Cloud
- **モデル名称**: AI Adoption Framework
- **業界**: 業界非依存
- **レベル数**: 3フェーズ
- **各レベルの定義**:
  1. **Tactical**: 単純で短期的なユースケース。Quick Win、低リスク導入
  2. **Strategic**: 持続的な事業価値を創出。ML本番運用、拠点横断展開
  3. **Transformational**: AIがイノベーションと業務俊敏性のエンジン。実験文化が定着
- **評価軸**: People / Process / Technology / Data の4基盤 × 6テーマ（Learn / Lead / Access / Scale / Secure / Automate）で評価
  - ① 業務プロセス統合度: 単一ユースケース → 複数業務 → 業界変革
  - ② 組織展開度: 一部の専門家 → 部門横断 → 全社+生態系
  - ③ 自動化レベル: 部分手動 → MLOps自動化 → エンドツーエンド継続学習パイプライン
  - ④ データ活用度: 局所データ → エンタープライズDWH連携 → プラットフォーム化
  - ⑤ ガバナンス成熟度: 局所判断 → 中央分析チーム標準化 → 責任あるAIプラットフォーム化
- **特徴**: 3フェーズと6テーマのマトリクス（Cloud Maturity Scaleの派生）。シンプルで実用的
- **想定ユースケース**: クラウド移行とAI活用を統合的に進める企業

### 10. Accenture AI Maturity Index（AI Achievers）

- **提唱主体**: Accenture
- **モデル名称**: AI Maturity Index（"Art of AI Maturity"）
- **業界**: 16業界・15ヶ国の調査ベース
- **レベル数**: 4区分（連続スコア0–100でも評価）
- **各レベルの定義**:
  - **AI Experimenters (63%)**: 平均的な能力のみ、PoC段階
  - **AI Builders (12%)**: 基盤能力は強いが戦略が弱い
  - **AI Innovators (13%)**: 戦略は強いが基盤が弱い
  - **AI Achievers (12%)**: 戦略・基盤両方が強く、収益成長率50%高い
- **評価軸**: Foundational能力（クラウド/データ基盤/ガバナンス）× Differentiation能力（戦略/CXOスポンサーシップ/責任あるAI/タレント/文化）
  - ① 業務プロセス統合度: 実験 → 基盤あり → 戦略あり → 全社統合
  - ② 組織展開度: 個別実験 → 部門 → 戦略主導 → 全社
  - ③ 自動化レベル: 試行 → 部分運用 → 戦略運用 → AIコア化
  - ④ データ活用度: 整備中 → 基盤あり → 戦略連動 → AI Core プラットフォーム
  - ⑤ ガバナンス成熟度: 検討中 → ガイドラインあり → 責任あるAI設計 → 規制対応最適化
- **特徴**: 2x2マトリクスで分類が直感的。**収益成長率や AI-influenced revenue 30%超** といった財務KPIと連動
- **想定ユースケース**: 経営層向け、競合比較、投資判断

### 11. Deloitte AI Maturity Levels

- **提唱主体**: Deloitte（State of AI in the Enterprise レポート）
- **モデル名称**: AI Maturity Levels for Digital Value
- **業界**: 業界非依存（24カ国6業界調査ベース）
- **レベル数**: 4
- **各レベルの定義**:
  1. **Basic Automation**: 単発の自動化
  2. **Agent-based Processes**: 単一エージェントが業務を担当
  3. **Process Reimagination**: マルチエージェントでプロセス再構築
  4. **Organizational Redesign**: 組織自体をAI前提で再設計
  - 大括りで **Automators**（Lv1-2）と **Transformers**（Lv3-4）に分類
- **評価軸**:
  - ① 業務プロセス統合度: 単発 → 単一エージェント業務 → プロセス再構築 → 組織再設計
  - ② 組織展開度: 部門単位 → 業務横断 → 全社 → ビジネスモデル
  - ③ 自動化レベル: ルールベース → 単一エージェント → マルチエージェント → 自律組織
  - ④ データ活用度: 局所 → 業務横断 → リアルタイムAIバックボーン → ドメイン主導データプロダクト
  - ⑤ ガバナンス成熟度: 局所 → 標準化 → 上級リーダーシップ主導 → 全社統制
- **特徴**: Automators vs Transformers の二分法が経営層に刺さりやすい。エージェントAIのガバナンス成熟度は21%のみ、と現実が映る数字を提供

## 横断分析

### 1. レベル定義の類型化

調査した20モデルを進化軸で分類すると、以下の4つの類型に分けられる。

#### 類型A：個人 → 部門 → 全社型（組織展開軸主導）

**代表モデル**: Gartner, Microsoft（旧来型）, NRI AI共創モデル, フォーティエンス, IPA DX推進指標, MITRE

**特徴**: 「誰が使っているか」の広がりを主軸にする。経営層が現状を理解しやすく、実務的な対話の土台になりやすい。

**進化のロジック**: 個人スキルの積み上げ → 部門での体系化 → 全社展開 → エコシステム化

---

#### 類型B：PoC → 実運用 → 変革型（フェーズ進化軸）

**代表モデル**: Google Cloud（Tactical/Strategic/Transformational）, BCG（Deploy/Reshape/Invent）, NRI（ステージ1-3）, AWS（4段階）

**特徴**: 短期目標→中期目標→長期目標の時間軸変化。投資配分判断と整合しやすい。

**進化のロジック**: 短期Quick Win → 業務効率化 → ビジネスモデル変革

---

#### 類型C：分析 → 予測 → 自動化 → 自律型（技術成熟軸）

**代表モデル**: Salesforce Agentic AI, Microsoft Agentic AI, KPMG（Lv0-5）, Deloitte（Lv1-4）

**特徴**: AI技術の自律度の進化を軸にする。エンジニアリング組織や技術投資判断に有用。

**進化のロジック**: ルールベース → 推奨 → 単一エージェント → マルチエージェント → 自律組織

---

#### 類型D：能力多軸×成熟度レベル（マトリクス型）

**代表モデル**: 日立 MA-ATRIX（7軸×7段階）, MITRE（6柱×5レベル×20次元）, Gartner（7柱×5レベル）, PwC（5レバー×5レベル）

**特徴**: 多次元評価で能力ギャップを可視化。実務的な改善計画策定に最も使いやすい。

**進化のロジック**: 各軸を独立に成熟させながら全体最適を図る

---

#### 類型E：行動・文化型（ソフトスキル軸）

**代表モデル**: Asana × Anthropic 5C Framework, Accenture（Foundational vs Differentiation）

**特徴**: 「AIを道具として見るか、チームメイトとして見るか」のような、人間側の心理・文化変化を中心に置く。

**進化のロジック**: 懐疑 → 試行 → 協働 → 戦略統合 → 文化定着

### 2. 業界差異の分析

#### 業界による成熟度定義の違い

| 業界 | 重視される成熟度の側面 | 代表的なモデルアプローチ |
|---|---|---|
| **金融** | ガバナンス・コンプライアンス・説明可能性 | PwC, MITRE, GSA |
| **製造業** | フィジカルAI・現場OT統合・予兆保全 | 日立 MA-ATRIX, Hitachi Lumada, BCG |
| **公共・政府** | 倫理・公平性・市民への説明責任 | MITRE, GSA AI CMM, EU AI Act 連動 |

| **小売・CX** | 顧客体験変革・パーソナライゼーション | Salesforce, Accenture |
| **IT/SI** | エンジニアリング能力・MLOps/GenAIOps | Microsoft GenAIOps, KPMG, IBM |
| **コンサル/プロサ** | 知識労働者の協働、ナレッジ統合 | Asana × Anthropic, NRI, McKinsey |

#### 共通している要素（業界依存しない本質的成熟度要素）

すべての業界・モデルで共通して登場する成熟度要素は以下：

1. **戦略・経営スポンサーシップ**: AIが戦略アジェンダに組み込まれているか
2. **データ基盤**: データ品質・統合・アクセス
3. **人材・スキル**: AIリテラシー・専門人材確保
4. **ガバナンス・倫理**: 責任あるAI、コンプライアンス
5. **組織・文化**: 実験文化、変革受容性
6. **業務プロセス統合**: 業務の中にAIが組み込まれている度合い
7. **計測と改善**: KPI・ROI計測、フィードバックループ

#### 業界依存しない本質的成熟度要素（再構成）

これら共通要素を、5軸に再整理：

```
①業務プロセス統合度
  → AIがどれだけ「業務の文脈」に組み込まれているか
  → 全業界共通の本質要素

②組織展開度
  → 個人・小規模 → 部門 → 全社 → エコシステム
  → 全業界共通

③自動化レベル
  → 支援 → 部分自動化 → 自律化
  → 業界によって「どこまで自律化を許容するか」は異なるが軸は共通

④データ活用度
  → 散在 → 整備 → 統合 → リアルタイムAIバックボーン
  → 全業界共通

⑤ガバナンス成熟度
  → ルールなし → 局所ルール → 全社標準 → 継続改善
  → 業界の規制要件で深さは異なるが軸は共通
```

## 統合AI成熟度モデル（業界非依存）

ここまでの20モデルの分析を統合し、**業界・規模を問わず適用可能な、実務に直結する統合AI成熟度モデル**を以下に提案する。

### 設計方針

1. **5段階レベル**（経営層に説明しやすい）×**5評価軸**（多次元診断）
2. **各軸×レベル**ごとに**KPI**を明確化（実測可能）
3. 国内外の20モデルから本質的要素のみを抽出し、**冗長性を排除**
4. **生成AI/エージェントAI時代**を前提とした設計（古いML中心モデルではない）

---

### 5段階レベルの定義

| レベル | 名称 | 一言定義 | 典型企業の状態 |
|---|---|---|---|
| **L1** | Awareness（認識） | AIへの関心はあるが、組織として活用なし | 経営層が議題に上げるが、実装活動なし |
| **L2** | Pilot（試行） | 個人・部署単位の試行・PoC実施 | ChatGPT Enterpriseアカウント配布、個別の実験 |
| **L3** | Operational（運用） | 一部業務でAIが本番運用、効果計測あり | 業務プロセスに組み込まれたユースケースが複数稼働 |
| **L4** | Scaled（全社展開） | 全社レベルでAIが業務不可欠。ガバナンス確立 | AI CoE機能、横展開が標準化 |
| **L5** | Transformational（変革） | AI起点の事業変革、エコシステム連携 | 新規ビジネスモデル、顧客接点の根本変革 |

> 補足：日立 MA-ATRIX 流に Lv0「未着手」と Lv6「自律最適化」を加える拡張も可能だが、実用上は5段階で十分。

---

### 5評価軸 × 5レベル マトリクス

#### ① 業務プロセス統合度

| Lv | 状態 |
|---|---|
| L1 | 業務とは無関係。試用のみ |
| L2 | 個人作業の補助（メール文案、要約等） |
| L3 | 1–3業務プロセスに組み込み、KPIで効果計測 |
| L4 | 主要業務プロセスの過半数にAIが組み込まれている |
| L5 | 業務プロセスがAI起点で再設計され、AI前提の新業務が誕生 |

#### ② 組織展開度

| Lv | 状態 |
|---|---|
| L1 | 一部のエンスージアストのみ |
| L2 | パイロット部門、または個別チーム単位 |
| L3 | 複数部署で運用、CoE機能が立ち上がっている |
| L4 | 全社員の50%以上が日常業務でAI活用 |
| L5 | 全社員の80%以上が活用、サプライヤ・顧客とのエコシステム連携 |

#### ③ 自動化レベル

| Lv | 状態 |
|---|---|
| L1 | 自動化なし |
| L2 | AIが個人の作業を「支援」（人間が必ず判断） |
| L3 | 一部業務でAIが「部分自動化」（人間が承認） |
| L4 | 業務プロセスが「条件付き自律実行」（重要判断のみ人間） |
| L5 | マルチエージェントによる「自律実行」、人間は監督・例外対応 |

#### ④ データ活用度

| Lv | 状態 |
|---|---|
| L1 | データがサイロ化、品質保証なし |
| L2 | パイロット用データを整備、限定的アクセス |
| L3 | 業務単位のデータ統合、データカタログあり |
| L4 | 全社データプラットフォーム、リアルタイム連携 |
| L5 | 全社×顧客×サプライヤ連携の "Living Data Backbone"。プロプライエタリデータが競争優位 |

#### ⑤ ガバナンス成熟度

| Lv | 状態 |
|---|---|
| L1 | ルールなし、シャドーAI状態 |
| L2 | 利用ガイドラインを文書化、教育開始 |
| L3 | AIガバナンス委員会設置、リスクアセスメント実施 |
| L4 | Responsible AI フレームワーク全社実装、第三者監査受入可 |
| L5 | 自律監視・継続改善、規制改正に先行対応、業界標準への寄与 |

---

### 各レベル判定のための具体的KPI

実務で「いまLnか」を判定するためのKPIを以下に整理。**3軸以上で該当レベルを満たすこと**を判定基準とする。

#### L1 → L2 移行のKPI（Awareness → Pilot）

| 軸 | KPI | 閾値 |
|---|---|---|
| 業務統合 | AIツール導入数 | 1以上（ChatGPT/Copilot等） |
| 組織展開 | AI利用社員比率 | 5%以上 |
| 自動化 | 自動化された個別タスク数 | 1以上 |
| データ | データ整備プロジェクト数 | 1以上 |
| ガバナンス | AI利用ガイドライン文書 | 草案あり |

#### L2 → L3 移行のKPI（Pilot → Operational）

| 軸 | KPI | 閾値 |
|---|---|---|
| 業務統合 | 本番運用中のAIユースケース数 | 3以上 |
| 業務統合 | 業務時間削減実績 | 計測されている（例：月100時間削減） |
| 組織展開 | AI利用社員比率 | 30%以上 |
| 組織展開 | AI CoE 有無 | あり（兼任でも可） |
| 自動化 | 業務プロセスへの自動化組み込み数 | 2以上 |
| データ | データカタログ整備率 | 主要業務データの50%以上 |
| ガバナンス | AIリスクアセスメント実施数 | 全AIユースケースに対して実施 |
| ROI | AI投資回収率 | 計測体制あり、初期ROI算出済 |

#### L3 → L4 移行のKPI（Operational → Scaled）

| 軸 | KPI | 閾値 |
|---|---|---|
| 業務統合 | 主要業務でAI組込率 | 50%以上 |
| 業務統合 | PoC→本番化率 | 40%以上（Deloitte 2026調査の上位企業基準） |
| 組織展開 | AI利用社員比率 | 60%以上（週次利用） |
| 組織展開 | AI専門人材数 / 全社員 | 1%以上 |
| 自動化 | 自律エージェント運用数 | 1以上（条件付き自律） |
| データ | 全社データプラットフォーム | 構築済み |
| データ | リアルタイムデータ連携 | 主要業務で実現 |
| ガバナンス | Responsible AI フレームワーク | 全社展開 |
| ガバナンス | AI監査体制 | 内部監査を実施 |
| ROI | AI由来の収益・コスト改善 | 営業利益の3%以上に寄与 |

#### L4 → L5 移行のKPI（Scaled → Transformational）

| 軸 | KPI | 閾値 |
|---|---|---|
| 業務統合 | AI起点の新業務・新サービス数 | 1以上（事業として独立） |
| 組織展開 | AI利用社員比率 | 80%以上 |
| 組織展開 | エコシステム連携（顧客・パートナー） | 実装済 |
| 自動化 | マルチエージェントの自律オーケストレーション | 主要業務で運用 |
| データ | プロプライエタリデータの戦略的活用 | 競争優位として機能 |
| ガバナンス | 規制対応の業界標準への寄与 | 業界団体・標準化への参画 |
| ROI | AI-influenced revenue | 全社売上の30%以上（Accenture基準） |

## AI CoE設計における活用方法

### 統合モデルとAI CoEの対応関係

AI CoE（Center of Excellence）の役割・規模・組織形態は、**現時点の組織のAI成熟度レベル**と、**目指すレベル**によって設計する必要がある。

#### Step 1: 現状診断（As-Is）

統合5軸×5レベルのマトリクスで自社（または対象組織）を診断する。**各軸独立に評価**し、「最も低い軸」がボトルネックになる（最も高い軸ではない）。

**診断のための質問例**：

- ① 業務プロセス統合度: 「過去6ヶ月で本番運用した AI ユースケースは何件か？」
- ② 組織展開度: 「全社員の何%が週1回以上 AI ツールを使用しているか？」
- ③ 自動化レベル: 「人間の承認なしに完了する AI ベースの業務は存在するか？」
- ④ データ活用度: 「主要業務のデータは AI からアクセス可能な状態に整備されているか？」
- ⑤ ガバナンス成熟度: 「AI 利用に関する全社ガイドラインがあり、運用されているか？」

#### Step 2: 目標レベル設定（To-Be）

12–24ヶ月後の目標レベルを軸ごとに設定。**全軸L5を目指す必要はない**（業界・規模・戦略により最適レベルは異なる）。

#### Step 3: AI CoE の役割・体制設計

現状レベルに応じて、AI CoE のミッションは大きく変わる。

##### L1–L2 段階の AI CoE（Awareness/Pilot 期）

**ミッション**: 経営合意形成 + 試行支援

| 機能 | 内容 |
|---|---|
| 戦略策定 | AIの可能性を経営層に説明、戦略策定支援 |
| ユースケース発掘 | 部門ヒアリング、業務棚卸し |
| パイロット支援 | ツール選定、パイロット実装の伴走 |
| ガイドライン整備 | AI利用ガイドライン草案、リテラシー教育 |
| **規模** | 兼任3–5名 |

##### L2–L3 段階の AI CoE（Pilot/Operational 期）

**ミッション**: PoC→本番化のスループット向上 + ガバナンス整備

| 機能 | 内容 |
|---|---|
| ユースケース管理 | パイプライン管理、優先順位付け |
| 共通基盤整備 | LLM API、ベクトルDB、RAG基盤 |
| MLOps/GenAIOps | 評価・モニタリング基盤 |
| ガバナンス運営 | AIリスクアセスメント、Responsible AI委員会 |
| 教育 | リテラシー研修、エンジニア育成 |
| **規模** | 専任5–15名（業種・企業規模による） |

##### L3–L4 段階の AI CoE（Operational/Scaled 期）

**ミッション**: 全社展開の標準化 + 効果最大化

| 機能 | 内容 |
|---|---|
| ハブ＆スポーク運営 | 中央CoE + 各事業部ローカルCoE |
| プラットフォーム提供 | 全社共通AI基盤、エージェント基盤 |
| 標準化 | 開発標準、評価標準、コスト管理 |
| 横展開支援 | ユースケースカタログ、再利用パターン |
| ROI計測 | 投資効果計測、経営報告 |
| 規制対応 | 第三者監査対応、コンプライアンス |
| **規模** | 専任20–50名（中央CoE）+ 各事業部に分散 |

##### L4–L5 段階の AI CoE（Scaled/Transformational 期）

**ミッション**: 事業変革のドライバー + 業界標準への寄与

| 機能 | 内容 |
|---|---|
| 事業創造支援 | AI起点の新事業企画、内製プロダクト化 |
| エコシステム構築 | 顧客・サプライヤ連携、業界標準化参画 |
| プラットフォーム事業化 | 内部CoEを外販事業化（NRI/日立モデル） |
| 自律組織運営 | AI が AI を改善する組織化 |
| **規模** | 戦略的中核組織として位置付け、事業化された場合は数百名規模 |

---

### 各成熟度レベルでよく陥る罠とその対策

| Lv | よくある罠 | 対策 |
|---|---|---|
| **L1** | 「AIで何ができるか」を経営層が理解せず、永遠にL1に留まる | 経営研修、他社事例見学、外部コンサル招聘 |
| **L2** | PoC疲れ（88%が本番化失敗）、一部の熱心な人だけが触る | KPIセット（PoC→本番移行率）、ユースケース選定プロセス標準化 |
| **L3** | 部分最適、サイロ化、シャドーAI | AI CoEのハブ機能強化、共通基盤統合、ガバナンス確立 |
| **L4** | 「全社展開したつもり」だが利用率が伸びない、効果計測されない | 利用率KPI、効果KPI、ローカルチャンピオン制度 |
| **L5** | エコシステム連携の規制リスク、データ利用倫理問題 | 規制動向の先取り、業界標準化への参画、第三者監査 |

---

### AI CoE 設計の具体的成果物（参考）

L2→L3 移行を狙う場合、AI CoE は以下の成果物を3–6ヶ月で整備することを推奨：

1. **AI 利用ガイドライン**（情報セキュリティ・著作権・社外共有制限など）
2. **AI ユースケース管理台帳**（ステータス・KPI・ROI付き）
3. **AI リスクアセスメントシート**（DPIA類似、3-5段階リスクスコア）
4. **AI 効果測定指標（KPI）テンプレート**（業務時間削減、品質改善、新規収益など）
5. **AI 教育プログラム**（全社員向けリテラシー、業務担当者向け活用、エンジニア向け実装）
6. **AI 共通基盤アーキテクチャ図**（LLM API、ベクトルDB、RAG、認証認可、ログ監査）
7. **AI ガバナンス組織設計**（CoE、リスク委員会、各事業部のローカルチャンピオン）
